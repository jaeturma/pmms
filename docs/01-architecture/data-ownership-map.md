# PMMS Data Ownership Map

**Status:** Draft Complete — Pending Domain and Stakeholder Validation
**Related:** [bounded-context-catalog.md](bounded-context-catalog.md) · [context-map.md](context-map.md) · [phase-0.2-domain-architecture.md, Section 8](phase-0.2-domain-architecture.md#8-data-ownership-principles)

## Principles

1. Every major data concept has exactly **one** authoritative (owning) context.
2. Other contexts may hold **references, projections, caches, or snapshots** of that data — never a second authoritative copy.
3. Public views (BC-29) are never authoritative, regardless of how complete they appear.
4. Reporting/analytics (BC-33) is never a transactional source of truth.
5. Corrections are made by the authoritative owner only, following that context's controlled correction pattern (see [high-integrity-domain-rules.md](high-integrity-domain-rules.md)) — never by a downstream consumer patching its own copy.
6. Sensitivity, retention importance, and public-exposure classification are set by the authoritative owner and must be respected by every consumer.

## Notes on Blocked Ownership Assignments

Several rows above show the authoritative *context* is settled but the specific *steward role* or the underlying *rule content* is blocked pending Phase 0.1 open decisions (eligibility authority, sports rules, result approval chain, medal tally rules). This is intentional: **bounded context ownership (technical/architectural boundary) is a Phase 0.2 decision; the human role and rule content within that boundary is a Phase 0.1 (policy) or Phase 0.3 (role/permission) decision.** Phase 0.2 does not need to wait for those answers to establish that, for example, "Eligibility and Clearance is the one and only authoritative owner of eligibility decisions" — it only needs those answers before the *workflow inside that boundary* can be finalized.
