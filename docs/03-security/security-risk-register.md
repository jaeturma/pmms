**Status:** Draft Complete — Pending Security, Privacy, Legal, Audit, Data Governance, and Engineering Validation
**Related:** [threat-model.md](threat-model.md) · [compliance-control-framework.md](compliance-control-framework.md) · [../00-product/assumptions-constraints-risks.md](../00-product/assumptions-constraints-risks.md)

This register tracks security/privacy/data-governance risks using Risk ID prefix `SR-` (Security Risk), distinct from Phase 0.1's product-level `RSK-` register and Phase 0.3's `SOD-` conflict matrix. **No numerical likelihood, impact, or risk rating is invented** — every such field is a placeholder pending a formal risk-assessment exercise, per working rule ("Do not invent numerical ratings").

## Risk Register Governance

- **No numerical rating is invented.** Every Impact/Likelihood/Inherent Risk/Residual Risk cell above is a `Placeholder` pending a formal risk-assessment exercise with a named methodology and named risk owner — this register establishes the register's *structure*, not its scored content.
- **Every risk has a named owner category** (not a named individual), consistent with [security-architecture.md, Section 3](security-architecture.md#3-security-governance-model).
- **Every risk has a review trigger** — a specific event that should prompt re-assessment, not merely a calendar date.
- **This register is a living document** — new risks are added as they're identified; resolved/accepted risks are marked accordingly, never silently removed.
